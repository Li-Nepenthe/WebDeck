# WebDeck 控制台使用指南

本指南将帮助你快速启动并使用 WebDeck 演示项目管理台。

## 1. 环境准备 (配置 Node.js)

在运行本项目之前，需要确保你的电脑已经安装了 Node.js 环境：
1. 访问 Node.js 官网：[https://nodejs.org/](https://nodejs.org/)。
2. 下载并按照提示安装 LTS (长期维护) 版本。
3. 安装过程保持默认选项即可，一直点击“下一步”直到完成。

## 2. 启动控制台

无需手动输入复杂的命令行，只需执行以下简便操作：
1. 在当前项目文件夹下找到 `start.bat` 脚本文件。
2. **双击运行 `start.bat`**。
3. 脚本会自动检查环境、安装所需的依赖，并在浏览器中自动为你打开 WebDeck 控制台界面。

## 3. 控制台功能概览

成功进入控制台后，你可以使用以下主要功能来高效管理你的演示内容：

- **章节管理**
  - **创建章节**：在左侧区域点击新增章节（相当于演示的各个部分）。
  - **重命名与排序**：可以随时修改章节名称，并通过拖拽轻松调整章节的前后顺序。
  
- **幻灯片内容导入**
  - **拖拽上传**：选中某个章节后，将本地准备好的 HTML 幻灯片文件直接拖入中间的内容区域即可快速导入。

- **项目预览与放映**
  - **实时预览**：点击预览按钮，可以直接在浏览器中查看当前的演示效果，管理台的修改会实时同步。

- **离线交付与导出**
  - **一键导出**：点击右上角的“导出离线项目”，系统会将当前所有的幻灯片合并打包到 `dist` 目录下的独立文件夹中。
  - **便于分享**：导出的离线项目已解除了所有环境依赖。你可以直接将该文件夹打包发送给其他人，对方只需双击里面的 `index.html` 就能直接在任意浏览器播放演示，支持完全断网脱机使用。
  - **目录管理**：支持一键打开 `dist` 目录或清空旧的导出结果。

## 🤖 WebDeck HTML 代码规范

将以下规范发送给任意 AI 助手，即可确保其生成的 HTML 代码兼容 WebDeck 框架。具体的演示内容、章节结构、页面数量等由你在对话中自行决定。

```markdown
**【WebDeck 幻灯片 HTML 代码规范】**

你生成的每个 HTML 文件都将被导入 WebDeck 演示框架。请严格遵守以下技术规范，确保代码能在框架中正确渲染。

---

### 📦 输出格式

你只需输出 **HTML 代码片段**。框架会自动在外层包裹 `<html>/<head>/<body>` 以及 `.slide > .slide-content` 容器结构。

⚠️ **绝对不要包含** `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>` 标签！

---

### ✅ `<style>` 标签使用规则

你 **可以且应该** 在每个页面开头使用 `<style>` 标签来定义该页面的专属 CSS 样式。所有自定义样式 **必须使用 class 选择器**作为作用域，绝对不能使用全局标签选择器。

**正确示例：**

    <style>
      .my-section { display: flex; gap: 2rem; }
      .my-section .card {
        padding: 1.5rem;
        background: var(--glass-bg);
        border: 1px solid var(--glass-border);
        border-radius: 8px;
      }
      .my-section .card h3 { color: var(--accent-color); }
    </style>

---

### 🚫 CSS 安全红线（违反任何一条都会破坏框架）

- 严禁 `*` 通配符选择器
- 严禁 `html`, `body` 标签选择器
- 严禁使用框架保留选择器：`#top-nav`, `.slide`, `.slide-content`
- 严禁 `!important` 声明
- 严禁 `position: fixed`
- 严禁直接裸写标签选择器如 `h1 { }`, `p { }`, `li { }`（必须加自定义类前缀，如 `.my-page h1 { }`）

---

### 🎨 框架内置设计系统

#### 可用 CSS 变量（直接在 `<style>` 中引用）

| 变量名 | 值 | 用途 |
|--------|-----|------|
| `--accent-color` | `#005eaa` | 主题蓝 |
| `--accent-light` | `#38bdf8` | 亮蓝色 |
| `--text-primary` | `#0f172a` | 主要文字色 |
| `--text-secondary` | `#475569` | 次要文字色 |
| `--glass-bg` | `rgba(255,255,255,0.85)` | 毛玻璃背景 |
| `--glass-border` | `rgba(0,94,170,0.15)` | 毛玻璃边框 |
| `--shadow-color` | `rgba(0,94,170,0.08)` | 阴影色 |

#### 框架已自动生效的排版（无需手动设置）

- `<h1>` → 4rem，蓝色渐变文字，font-weight 800
- `<h2>` → 2.5rem，学术蓝纯色，font-weight 600
- `<p>`, `<li>` → 1.5rem，灰色正文，1.6 行高
- `<ul> > <li>` → 自带蓝色 ▪ 列表符号

#### 封面页专用类名

当页面位于 `01_封面` 文件夹时，框架会自动隐藏导航栏并启用全屏布局。封面页可使用以下框架预设类名：`.cover-page` > `.title-container` + `.author-info`（含 `.presenter`, `.university` 子类）。

---

### 📐 排版建议

- 每页信息要点建议不超过 5~7 条
- 善用可视化布局（卡片、双栏、表格、高亮框等），避免纯文本堆砌
- 每一页都应有设计感的视觉层次，不要只是裸写 `<h1>` + `<ul>`
- 数学公式：框架原生支持 MathJax，行内 `\( ... \)`，块级 `$$...$$`
- 占位图片：`<img src="图片占位.png" style="width: 80%; border-radius: 8px;">`
```
